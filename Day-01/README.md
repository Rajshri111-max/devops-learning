## 🧠 My Understanding

### ❓ Why is Internet & Networking important in DevOps?

As a DevOps engineer, you help developers deploy applications on servers.
The Internet and networking act as a **communication bridge** between systems.

In DevOps:

* 💻 Applications are deployed on servers
* 🌍 Users access them through the Internet
* 🔄 Services communicate with each other

👉 Everything in DevOps works on top of Internet and networking

## 🌐 Internet

The Internet is a **network of networks** connected globally.

---

## 🔗 Network

A network is a **group of devices connected together to communicate and share data**.

---

## 📡 Protocols

Protocols are **rules that define how data is transferred between devices**.

### 🔹 Types of Protocols:

* **IP (Internet Protocol):**
  Provides a unique address to each device for communication

* **TCP (Transmission Control Protocol):**
  Ensures data is delivered **accurately and in order**

* **UDP (User Datagram Protocol):**
  Used for real-time communication (video calls, voice calls)
  Faster but less reliable

* **HTTP (HyperText Transfer Protocol):**
  Transfers data between browser and server (not secure)

* **HTTPS (HyperText Transfer Protocol Secure):**
  Secure version of HTTP using encryption (SSL/TLS)

---

## 📦 Packet Switching

Packet switching is used for all Internet communication.

### 🔍 How it works:

* Data is divided into small packets
* Packets travel through different paths
* Reassembled at the destination

### 📖 Example:

Imagine sending a book:

* Tear it into pages
* Number each page
* Send separately

Receiver:

* Collects all pages
* Arranges them in order

👉 This is how packet switching works
